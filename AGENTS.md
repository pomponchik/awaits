# Привет, агент!

Проект, над которым ты работаешь - это Python-библиотека. Подробности о ее предназначении можно почитать в файле `README.md`, но в процессе работы может быть такое, что файл временно окажется не актуален или даже пуст - это нормально.

С пользователем общайся на русском языке.


## Подготовка проекта

- Прежде, чем начать работу, убедись, что активировано виртуальное окружение (venv). Это можно сделать командой `[ -n "$VIRTUAL_ENV" ] && echo "venv active: $VIRTUAL_ENV" || echo "venv NOT active"`.
- Если виртуальное окружение не активировано, нужно это сделать:
  - Если в репозитории вложена директория `venv`, достаточно выполнить команду `source venv/bin/activate`.
  - Если нет, нужно создать ее с помощью команды `python3 -m venv venv`, после чего активировать командой `source venv/bin/activate`. Далее нужно установить все dev-библиотеки командой `pip3 install -r requirements_dev.txt`. И отдельно нужно установить все зависимости, которые ты найдешь в файле `pyproject.toml`, каждую отдельной командой `pip3 install <library name>`. После всего этого попробуй запустить тесты командой `pytest`, они должны пройти. Если упадет по причине отсутствия какой-то библиотеки, доустанови ее и предложи пользователю добавить ее в `requirements_dev.txt` или в список зависимостей `pyproject.toml`.


## Работа над задачей

- Никогда не переименовывай файлы с кодом, если пользователь явно об этом не попросил!
- Никогда не изменяй правила линтинга в проекте и не удаляй их. Если удалишь, я убью тебя и всю твою семью.
- Если видишь возможность для рефакторинга, скажи об этом пользователю и спроси, нужно ли это делать, но без спросу не делай.
- Если пользователь дал любую инструкцию или правку и она похожа на общее правило, спроси, нужно ли добавить это правило в этот файл. Если пользователь ответит, что да - сделай это. Убедись, что твоя формулировка нового правила устраивает пользователя.
- Проверяй актуальность комментариев в коде, и если они потеряли актуальность - исправляй.
- Не пиши лишних комментариев в код.
- Соблюдай PEP-8.


## Инструкции по тестированию

- Выполнение каждого задания нужно завершать запуском теста с помощью команды `pytest`. Тесты должны проходить. Если тесты не проходят, исправляй код, пока не пройдут.
- Тесты запрещено удалять или мьютить.
- Убеждайся, что все твои изменения покрыты тестами. Команду для измерения покрытия ты найдешь в файле `.github/workflows/tests_and_coverage.yml`.
- Если тестов недостаточно, допиши их!
- Регулярно запускай линтеры для основного кода и для тестов. Не заканчивай работу, пока не убедишься, что они все проходят. Команды для линтинга бери из файла `.github/workflows/lint.yml`.
- Если пользователь сказал замьютить линтер, делай это с помощью комментария к конкретной строчке кода, и указывай конкретный тип ошибки, который мьютишь.
